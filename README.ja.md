# Vault Alchemist

**📖🧠🖥️ — 「Obsidianを、AIと融合した"資産運用"の書斎に変える」**

---

## なぜ、このツールを作るのか

情報は増え続けている。

毎日、何かを読み、考え、誰かと話し、AIと議論する。
その痕跡はメモになり、PDFになり、会話ログになる。

でも、気づけば**使えていない**。

タグは増えすぎて崩壊し、ノートは探せず、
AIに渡しても「文脈が足りない」と言われ、
昨日すごく重要だと思ったことが、今日はどこにあるか分からない。

> 見つけられないデータは、ないも同じだ。

---

## Vault Alchemistとは何か

Vault Alchemistは、Obsidianプラグインだ。

でも、それだけではない。

これは**「思考と対話を資産に変えるエンジン」**だ。

AIが「図書館司書＋編集者」として、あなたのVaultを静かに整理する。
長すぎる本は「章」に分け、断片メモは「テーマ別ノート」にまとめ、
価値が薄いものは「候補」として提示し、
人が見て**「今読む価値」が一目でわかる表紙**を付ける。

そして、すべては**あなたの承認なしには変わらない**。

---

## 体験としての約束

このプロジェクトは、技術的な目標より先に、**体験の約束**を持っている。

### 対話は流れない
チャット履歴は流れていく。
今日Claude と議論した設計の話も、来週には「どこかの会話」になる。

Vault Alchemistは、それを「話題ごとの知識」に再構成する。
ChatGPTとの会話も、Claudeとのやり取りも、全部、**後から再利用できる形**に変わる。

### AIは整理まで。決めるのは人間
破壊的変更——本文の書き換え、削除、大規模な移動——は、**必ず人間の承認を通す**。

「夜中にこっそり整理」はやる。でも、適用するのはあなただ。

### 安心が最優先
日記、手紙、メール、家族との会話——これらは「感情資産」だ。
どんな整理も、感情資産を壊してはいけない。

Vault Alchemistはこれらを「編集不可」として認識し、
代わりに**表紙（要約・行動ログ）を足す**ことで検索性を高める。
本文には触れない。

### 同じ作業を二度させない
ハッシュで変更を追跡し、差分だけ更新する。
失敗しても途中から再開できる。
ロールバックはいつでも、ワンボタンで。

### ローカル優先
あなたのデータは、あなたのPCにある。

外に出るのは、あなたが意図して選んだときだけ。

## 「Persona公開モード」——人と人が理解し合うために

Vault Alchemistには、将来「Persona公開モード」が加わる。

これは、**あなたの思想・構想・知識の一部を、MCP経由で外部に公開する**機能だ。

たとえば——

起業家が「プロダクトビジョン」ノートを公開すると、  
投資家や共感者がAIを通じて**その人の考え方と深く対話できる**。

政治家が「政策思想」を公開すると、  
国民がAIを通じて議論し、**より深く理解し合える**。

そして、その対話のデータは——  
本人の許可があれば——**再びVaultに蓄積される**。  
対話そのものが資産になる。

重要なのは、これが「本人」と「本人の知識を元にしたAI」を区別することだ。

> **「イーロン・マスク」と「AI-Elon（イーロンの公開知識による）」は、最初から別物として明示する。**

なりすましの完璧な解決は目指さない。ハルシネーションと同じく、「明確な区別」を設計に組み込む。

このプロジェクトの最終的な夢は——  
**多くの人が、お互いをもっと深く理解し合えるようになること**だ。

## 設計の哲学

### AIは道具ではなく、協働者
AIに「書かせる」のではなく、AIと「一緒に整理する」。
人間は意図を持ち、AIは整理の力を提供する。

### デフォルトは安全
初期状態では何も壊さない。dry-run（試運転）から始め、
ユーザーが「信頼した分だけ」自動化が進む。

### 判断を先送りできる
「削除候補」は即座に消さない。バッチにまとめて、後でまとめて判断する。
「急かさない」こともUXの一部だ。

### 言葉は体験を語る
このプロジェクトでは、技術用語でユーザーを驚かせない。

| ❌ 言わない | ✅ 代わりに言う |
|---|---|
| 「バックグラウンドプロセスを起動します」 | 「作業をバックグラウンドで行います」 |
| 「インデックスを再構築しています」 | 「検索を育てています」 |
| 「エラーが発生しました」 | 「うまくいかないアイテムがあります」 |

> Vault Alchemistは、あなたのじゃまにならないよう、裏側で静かに動いています 🌙

## オープンソースである理由

このプロジェクトはMITライセンスで、完全にオープンだ。

理由は一つではない。

まず、**「使ってみた人が広げてくれる」**という設計思想がある。
Obsidianユーザーはリテラシーが高い。彼らに実際に使ってもらい、
動いてから口コミで広がる——そのサイクルが最も自然だ。

次に、**AIと一緒に開発することが前提**だ。
コードベースが1つのリポジトリに収まっていると、
AIが全体のコンテキストを把握したまま作業できる。
「分割リポジトリ」はチームには向くが、AI協働開発には不向きだ。

そして、**このツールの使命は個人を超えている**。
知識を資産に変え、人と人が理解し合うインフラになるためには、
一人では作れない。

**コントリビューターを歓迎する。人間でも、AIでも。**

## 使い始めるには（コントリビューター向け）

```bash
git clone https://github.com/matsulinks/vault-alchemist
cd vault-alchemist
npm install
```

必要環境: Node.js 18以上、Obsidian 1.0以上

インストールするだけでserviceが自動起動する。  
「バックグラウンドで何かを起動する」操作は不要だ。

## 作者から、ともに歩くあなたへ

一緒に開発してくれる人へ、使ってくれる人へ、
そしてこのREADMEをたまたま読んでいるあなたへ。

僕は、どこにでもいる普通の人間だ。

欲もある。意地汚さも、卑怯さも、いやらしさも、当然に持ち合わせている。
清廉に生きたいと思いながら、気づけばブレていることがある。

だからこそ、この仕様を作り込んだ。

「データは壊さない」「承認なしに変えない」「感情資産を守る」——
これらはすべて、道具としての機能であると同時に、
**自分がブレたときに立ち戻るための杭**だ。

設計書に哲学を書いたのは、機能の説明のためだけじゃない。
未来の自分が「なぜこう作ったか」を思い出すためでもある。

だから頼みたいことがある。

コントリビューターの人も、ユーザーの人も——
このプロジェクトの方向が、書いてある哲学とズレていると感じたら、
**ちゃんと指摘してほしい。**

遠慮はいらない。それがこのプロジェクトを健全に保つことになる。
